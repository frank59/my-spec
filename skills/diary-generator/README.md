# Skill: 日记生成器 (Diary Generator)

## 概述

将 Obsidian Inbox 中的原始记录转换为结构化的每日日记，并关联到已有知识库。

---

## 输入

### 输入文件位置
```
{VAULT_PATH}/00-Inbox/YYYY-MM-DD.md
```

### 输入格式示例
```markdown
- 9:00 重构了订单模块的并发处理逻辑，用了读写锁
- 14:00 和架构组讨论缓存策略，决定用 Redis + 本地缓存二级架构
- 想法：是不是可以把这部分抽象成一个通用组件？
- 看了篇关于领域驱动设计的文章，觉得我们的项目可以借鉴聚合根的概念
- 收获：读写锁在读多写少场景下性能提升很明显，要记下来
```

### 输入规则
- 每行以 `- ` 开头
- 可选的时间前缀：`HH:MM `
- 内容类型通过前缀词识别：
  - 无前缀 → 「做的事」
  - `想法：` → 「想法」
  - `收获：` → 「收获」
  - `读了/看了` → 「学习输入」

---

## 处理步骤

### Step 1: 解析与分类
将原始记录按类型分类：
- **做的事**: 具体完成的工作、代码、会议等
- **想法**: 灵感、问题、假设
- **收获**: 学到的知识、验证的结论
- **学习输入**: 阅读的文章、论文、文档

### Step 2: 关键词提取与关联
识别技术关键词，建立与知识库的链接：
- 扫描 02领域/01计算机/ 目录下的现有笔记
- 匹配技术术语（如：Redis、并发、DDD、微服务等）
- 生成 Wikilink：[[技术笔记名称]]

### Step 3: 内容提炼
为每个类别生成一句话总结，突出关键信息。

### Step 4: 生成输出
按输出模板格式化，写入日记目录。

---

## 输出

### 输出文件位置

#### 当前月份日记（活跃）
```
{VAULT_PATH}/05日记/YYYY-MM-DD.md
```

#### 归档日记（次月自动归档）
```
{VAULT_PATH}/05日记/YYYY/MM/YYYY-MM-DD.md
```

**归档规则**:
- 当月日记：直接存放在 `05日记/` 根目录，便于快速访问和编辑
- 跨月归档：每月 1 日自动将上月日记移动到 `05日记/YYYY/MM/` 目录
- 归档后保持原有 Wikilink 链接有效

### 输出格式模板
```markdown
---
created: YYYY-MM-DD HH:mm
tags: 日记
date: YYYY-MM-DD
---

## 今日概览
一句话总结今天的主要活动和收获。

## 做的事
- [时间] 具体事项 → 关联笔记

## 想法
- 灵感/问题描述

## 收获
- 学到的知识点

## 学习输入
- 阅读内容 → 关键收获

## 明日关注
- [ ] 从今日事项中延伸的待办

## 相关链接
- [[相关笔记1]]
- [[相关笔记2]]
```

## 执行命令

### 生成今日日记
```bash
# 读取 Inbox 今日记录，生成日记
qoder skill diary-generator --date today
```

### 生成指定日期日记
```bash
qoder skill diary-generator --date 2026-04-02
```

### 批量生成（补录）
```bash
qoder skill diary-generator --from 2026-03-01 --to 2026-04-02
```

### 手动归档
```bash
# 归档指定月份日记到 YYYY/MM/ 目录
qoder skill diary-generator --archive 2026-03

# 归档上个月
qoder skill diary-generator --archive-last-month
```

## 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `VAULT_PATH` | Obsidian Vault 路径 | `/Users/wangheng/work/projects/gitee/obsidian` |
| `AUTO_LINK` | 是否自动关联知识库 | `true` |
| `SUMMARY_ENABLED` | 是否生成今日概览 | `true` |
| `ARCHIVE_ENABLED` | 是否启用自动归档 | `true` |
| `ARCHIVE_SCHEDULE` | 归档执行时间 | `每月1日 00:00` |

### 归档说明
- 归档操作将上月日记从 `05日记/` 移动到 `05日记/YYYY/MM/`
- 归档前自动更新所有相关 Wikilink
- 支持手动触发归档：`qoder skill diary-generator --archive 2026-03`

## 注意事项与限制

### 知识库关联
- **问题**: 自动生成的 Wikilink（如 `[[AI]]`、`[[领域驱动设计]]`）可能对应笔记不存在
- **解决**: 
  - 优先匹配已存在的笔记标题
  - 对于不存在的笔记，生成时标记为 `[[待创建: 领域驱动设计]]`
  - 定期 review 并创建缺失的笔记

### 时间格式
- **问题**: 输入时间格式不统一（`9:00` vs `09:00`）
- **建议**: 输出时统一格式化为 `HH:MM`

### 论文/文章信息
- **问题**: 论文名称、链接等信息在原始记录中不完整
- **建议**: 在 Inbox 记录时使用标准格式：
  ```
  - 看了《论文标题》(arXiv:xxx.xxxxx)：一句话概括
  ```
